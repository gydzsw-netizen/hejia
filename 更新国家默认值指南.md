# 更新国家运算规则默认值

本文档说明如何更新所有用户的国家运算规则默认值。

## 更新内容

### 所有国家统一参数
- 活动占比: 20%
- 广告占比: 10%
- 退款率: 5%
- 体积系数: 6000
- 体积运费系数: 8 元/kg
- 美元汇率: 7.0
- 欧元汇率: 7.8

### 各国家差异参数

| 国家 | 重量运费系数 | 固定成本 |
|------|-------------|---------|
| 🇺🇸 美国 | 55 元/kg | 6 元 |
| 🇩🇪 德国 | 80 元/kg | 25 元 |
| 🇬🇧 英国 | 80 元/kg | 25 元 |
| 🇫🇷 法国 | 78 元/kg | 31 元 |
| 🇪🇺 泛欧 | 90 元/kg | 60 元 |

## 执行步骤

### 方法 1: 使用本地脚本（推荐）

如果你有本地数据库访问权限：

```bash
# 1. 确保环境变量已配置
cp .env.example .env
# 编辑 .env 文件，填入数据库连接信息

# 2. 执行更新脚本
node execute-sql.js sql/update-country-defaults.sql
```

### 方法 2: 使用 Vercel Postgres 控制台

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 进入项目的 Storage 标签
3. 选择 Postgres 数据库
4. 点击 "Query" 标签
5. 复制 `sql/update-country-defaults.sql` 文件的内容
6. 粘贴到查询编辑器
7. 点击 "Run Query" 执行

### 方法 3: 使用 Vercel CLI

```bash
# 1. 安装 Vercel CLI（如果还没有）
npm i -g vercel

# 2. 登录
vercel login

# 3. 拉取环境变量
vercel env pull .env

# 4. 执行更新脚本
node execute-sql.js sql/update-country-defaults.sql
```

## 注意事项

1. **备份数据**: 执行更新前建议备份数据库
2. **测试环境**: 建议先在测试环境执行，确认无误后再在生产环境执行
3. **用户通知**: 更新后建议通知用户参数已更新
4. **保留自定义**: 脚本使用 `COALESCE` 函数，如果用户已自定义利润率和最低价格，将保留用户的设置

## 回滚

如果需要回滚，可以执行以下 SQL：

```sql
-- 回滚到旧的默认值（示例）
UPDATE user_settings SET
  us_activity_rate = 0.00,
  us_ad_rate = 0.00,
  us_refund_rate = 0.00,
  us_weight_rate = 10.00,
  us_fixed_cost = 5.00,
  us_usd_rate = 7.20;
-- ... 其他国家类似
```
