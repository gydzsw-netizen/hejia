## 部署步骤

### 步骤 1：数据库迁移

如果数据库还没有运行迁移脚本，请执行：

```bash
# 方法 1：使用 Vercel CLI
vercel env pull .env
node run-migration.js

# 方法 2：直接在 Vercel Postgres 控制台执行
# 1. 登录 Vercel Dashboard
# 2. 进入项目的 Storage 标签
# 3. 选择 Postgres 数据库
# 4. 在 Query 标签中执行 sql/add-country-settings.sql 和 sql/add-country-settings-part2.sql
```

### 步骤 2：部署到 Vercel

```bash
# 提交代码
git add .
git commit -m "完善多国家运费核算功能"
git push

# 部署到 Vercel（如果配置了自动部署，这一步会自动执行）
vercel deploy --prod
```

### 步骤 3：验证部署

1. 访问部署后的网站
2. 登录管理员账户
3. 进入价格核算页面
4. 按照《测试指南.md》进行测试

## 功能特性

### 用户功能
- ✅ 选择 5 个国家/地区（美国、德国、英国、法国、泛欧）
- ✅ 每个国家使用独立的运算规则
- ✅ 实时切换国家，自动加载对应规则
- ✅ 显示人民币、美元、欧元三种货币价格
- ✅ 详细的成本分解显示

### 管理员功能
- ✅ 为每个国家独立设置运算规则
- ✅ 自动保存设置到数据库
- ✅ 设置活动占比和广告占比（普通用户只读）
- ✅ 不同国家的设置互不影响

### 界面优化
- ✅ 国家选择器样式优化（边框、悬停效果、下拉箭头）
- ✅ 国旗图标显示
- ✅ 响应式设计，支持移动端
- ✅ 实时显示当前选择的国家

## 技术架构

### 前端
- HTML5 + CSS3
- 原生 JavaScript (ES6 Modules)
- 响应式设计

### 后端
- Vercel Serverless Functions
- Node.js
- JWT 认证

### 数据库
- PostgreSQL (Vercel Postgres)
- 用户设置表支持多国家字段

## 注意事项

1. **数据库迁移**：确保在部署前运行了所有迁移脚本
2. **环境变量**：确保 Vercel 项目配置了正确的数据库连接信息
3. **权限控制**：普通用户无法修改运算规则，只有管理员可以
4. **数据持久化**：所有设置都保存在数据库中，不会丢失
5. **浏览器兼容性**：支持所有现代浏览器（Chrome、Firefox、Safari、Edge）
