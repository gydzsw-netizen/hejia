## 参数说明

### 费率参数
- **活动占比**: 平台活动费用占销售价格的百分比
- **广告占比**: 广告推广费用占销售价格的百分比
- **退款率**: 预估退款损失占销售价格的百分比

### 运费参数
- **重量运费系数**: 每公斤实际重量的运费成本（元/kg）
- **体积系数**: 用于计算体积重量的系数（常用值 5000-6000）
- **体积运费系数**: 每公斤体积重量的运费成本（元/kg）
- **固定成本**: 包装、人工等固定成本（元）

### 汇率参数
- **美元汇率**: 1 美元 = X 人民币
- **欧元汇率**: 1 欧元 = X 人民币

## 计算公式

```
销售价格 = (产品成本 + 运费) / (1 - 利润率 - 活动占比 - 广告占比 - 退款率)

其中：
运费 = 重量运费 + 体积运费 + 固定成本
重量运费 = 实际重量 × 重量运费系数
体积运费 = 体积重量 × 体积运费系数
体积重量 = 长 × 宽 × 高 ÷ 体积系数
```

## 更新默认值

如果需要更新所有用户的默认值，执行以下 SQL 脚本：

```bash
node execute-sql.js sql/update-country-defaults.sql
```

或者在 Vercel Postgres 控制台中直接执行 `sql/update-country-defaults.sql` 文件。

## 管理员操作

管理员可以为每个国家独立设置运算规则：

1. 登录管理员账户
2. 进入价格核算页面
3. 选择目标国家
4. 在"运算规则"卡片中修改参数
5. 系统自动保存

## 注意事项

1. **不要删除国家选项**: 这 5 个国家是系统核心功能，删除会导致功能异常
2. **不要修改国家代码**: US、DE、GB、FR、EU 是固定代码，修改会导致数据库字段不匹配
3. **可以修改参数值**: 管理员可以根据实际情况调整各国家的运算规则参数
4. **参数独立存储**: 每个国家的参数独立存储，互不影响
