# 快速启动指南

## 本地开发测试

### 1. 安装依赖

```bash
cd C:/Users/旭光/hejia
npm install
```

### 2. 配置环境变量

创建 `.env` 文件（如果还没有）：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的数据库连接信息：

```
POSTGRES_URL="postgres://..."
POSTGRES_PRISMA_URL="postgres://..."
POSTGRES_URL_NON_POOLING="postgres://..."
JWT_SECRET="your-secret-key"
```

### 3. 运行数据库迁移

```bash
node run-migration.js
```

### 4. 启动本地开发服务器

```bash
npm run dev
```

服务器将在 http://localhost:3000 启动。

### 5. 访问系统

打开浏览器，访问：
- 登录页面：http://localhost:3000/index.html
- 主页：http://localhost:3000/home.html
- 价格核算：http://localhost:3000/calculator.html

### 6. 测试账户

如果还没有创建管理员账户，可以运行：

```bash
node execute-sql.js sql/create-admin.sql
```

默认管理员账户：
- 用户名：admin
- 密码：admin123

**重要**：首次登录后请立即修改密码！

## 常见问题

### Q1: 启动失败，提示端口被占用

**解决方法**：
```bash
# 修改端口
npm run dev -- -l 3001
```

### Q2: 数据库连接失败

**解决方法**：
1. 检查 `.env` 文件是否正确配置
2. 确认数据库服务是否正常运行
3. 检查网络连接

### Q3: 登录失败

**解决方法**：
1. 确认用户是否已创建
2. 检查密码是否正确
3. 查看浏览器控制台的错误信息

### Q4: 切换国家后运算规则没有变化

**解决方法**：
1. 确认数据库迁移是否成功执行
2. 检查浏览器控制台是否有错误
3. 清除浏览器缓存后重试

## 部署到 Vercel

### 方法 1：通过 GitHub 自动部署

1. 将代码推送到 GitHub：
```bash
git add .
git commit -m "完善多国家运费核算功能"
git push
```

2. 在 Vercel Dashboard 中：
   - 导入 GitHub 仓库
   - 配置环境变量
   - 点击 Deploy

### 方法 2：使用 Vercel CLI

```bash
# 安装 Vercel CLI（如果还没有）
npm i -g vercel

# 登录
vercel login

# 部署
vercel --prod
```
